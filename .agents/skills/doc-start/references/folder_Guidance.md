# Folder Guidance

Docs live in `Project_Manag/Docs/`. Use only the folders the repo actually needs.

| Folder | Use for |
|---|---|
| `Architecture` | Structure, systems, technical layout, integration boundaries |
| `Descr` | What the repo or product does, domain model, conceptual descriptions |
| `Features` | Feature-by-feature docs |
| `Decisions` | Important decisions, tradeoffs, ADR-like notes |
| `Investigation` | Research, comparisons, external analysis |
| `Learnings` | Lessons, gotchas, post-fact notes |
| `Live_Working` | Current work, open issues, task pools, active planning |
