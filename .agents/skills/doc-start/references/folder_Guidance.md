# Folder Guidance

Docs live in `Project_Manag/Docs/`. `doc_Start.md` itself lives at `Project_Manag/Docs/doc_Start.md` — not at the repo root. Use only the folders the repo actually needs.

| Folder | Location | Use for |
|---|---|---|
| `Architecture` | `Project_Manag/Docs/` | Structure, systems, technical layout, integration boundaries |
| `Descr` | `Project_Manag/Docs/` | What the repo or product does, domain model, conceptual descriptions |
| `Features` | `Project_Manag/Docs/` | Feature-by-feature docs |
| `Decisions` | `Project_Manag/Docs/` | Important decisions, tradeoffs, ADR-like notes |
| `Investigation` | `Project_Manag/Docs/` | Research, comparisons, external analysis |
| `Learnings` | `Project_Manag/Docs/` | Lessons, gotchas, post-fact notes |
| `Live_Working` | `Project_Manag/` | Current work, open issues, task pools, active planning — lives one level above Docs/, not inside it |
